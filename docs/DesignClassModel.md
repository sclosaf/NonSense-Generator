# Design Class Model
 The design class model was split into seven class diagrams to guarantee a better understandability of the graphs but still mantain a complete and punctual representation of the code. The diagrams can be found in the diagrams/classDiagrams directory; here is a brief description of each diagram.

## ClassDiagram_all
 This diagram represents the structure of the code, showing how each class and interface is connected and grouped up; variables, methods and exceptions are not represented.

## ClassDiagram_analyzer
 This diagram focuses on the analyzer package, displaying all the classes contained in it, each with its own methods and variables. The diagram also displays the dependencies the analyzer classes have on external classes, as well as the exceptions used.

## ClassDiagram_app
 This diagram focuses on the app package, displaying all the classes contained in it, each with its own methods and variables. The diagram also displays the dependencies the app classes have on external classes, as well as the exceptions used.

## ClassDiagram_exceptions
 This diagram focuses on the exception package, displaying all the exceptions used by the program, some of which contain a getter function. The exceptions are grouped up according to which type of main exception they extend.

## ClassDiagram_generator
 This diagram focuses on the generator package, displaying all the classes contained in it, each with its own methods and variables. The diagram also displays the dependencies the generator classes have on external classes, as well as the exceptions used.

## ClassDiagram_model
 This diagram focuses on the model package, displaying all the classes contained in it, each with its own methods and variables. The diagram also displays the dependencies the model classes have on external classes, as well as the exceptions used.

## ClassDiagram_util
 This diagram focuses on the util package, displaying all the classes contained in it, each with its own methods and variables. The diagram also displays the dependencies the util classes have on external classes, as well as the exceptions used.
